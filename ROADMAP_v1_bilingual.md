# Public Roadmap Summary (v1.0) | خلاصهٔ نقشه‌راه عمومی
**Author:** Tawana Mohammadi

**License:** CC BY 4.0

این خلاصهٔ عمومی، سانیتایز و غیرعملیاتی است و برای اطلاع‌رسانی در سطح بالا استفاده می‌شود.

## Key phases | فازهای اصلی
1) **Discovery / نیازسنجی:** Map user needs and risks at a high level; avoid infrastructure specifics.  
2) **Proposal / پیشنهاد:** Draft impact-focused proposals with ethical safeguards; keep technical details private.  
3) **Review / داوری:** Engage OTF/IFF for feedback, integrate comments, and confirm milestones and reporting cadence.  
4) **Execution / اجرا:** Track milestones and community impact using non-sensitive indicators; keep operational data off public channels.  
5) **Reporting / گزارش‌دهی:** Publish sanitized progress updates and link DOI-enabled releases on Figshare/Zenodo/ORCID.

## Safeguards | ملاحظات
- No domains, IPs, configs, or bypass instructions are recorded here.
- Weekly document hygiene: verify language clarity and remove any sensitive traces before sharing.
- Use secure channels for coordination and any credential handling.
