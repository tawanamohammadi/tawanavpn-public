# Tawanavpn / Tawana Proxy — Public Documentation | مستندات عمومی

This repository is the public, bilingual (Persian/English), sanitized, and non-operational home for the Tawanavpn roadmap materials. It organizes the full FA/EN articles, publication checklist, metadata, and license needed for Figshare/Zenodo or similar research archives. No configuration details, domains, IPs, or operational guidance are included.

این مخزن نسخهٔ عمومی، دوزبانه، سانیتایز و غیرعملیاتیِ مستندات Tawanavpn است. فایل‌های نقشه‌راه فارسی و انگلیسی، چک‌لیست انتشار، متادیتا و مجوز لازم برای آرشیوهای پژوهشی (Figshare/Zenodo) در اینجا قرار گرفته‌اند. هیچ جزئیات فنی، دامنه، IP یا راهنمای عملیاتی در این مخزن درج نشده است.

## Repository structure | ساختار مخزن
- `docs/articles/fa/Tawanavpn_Funding_Roadmap_v1_Version5.md`: Persian full article (v1, sanitized, non-operational).
- `docs/articles/en/ROADMAP_EN_v1.md`: English full article (v1, sanitized, non-operational).
- `docs/checklists/publication_and_sanitization_checklist.md`: Bilingual publication and sanitization checklist for documents, repos, and metadata.
- `metadata/figshare_metadata.json`: Example Figshare/Zenodo metadata (CC BY 4.0) for releases.
- `LICENSE`: Creative Commons Attribution 4.0 International license text.
- `CHANGELOG.md`: Public log of documentation updates (sanitized only).

## Scope and principles | دامنه و اصول
- Public, sanitized roadmap materials only; no operational circumvention details, configs, domains, IPs, or routing hints.
- Living roadmap updated weekly or at each phase milestone; updates remain high-level and non-operational.
- Publication assets (metadata, checklist, license) are provided to keep releases consistent and safe for sharing.

## Usage | استفاده
- Refer to the full FA/EN articles in `docs/articles/` for the current roadmap narrative and review workflow.
- Use the checklist in `docs/checklists/` before linking to any repository or uploading to Figshare/Zenodo to confirm sanitization.
- Pair Markdown sources with generated PDFs when publishing to research archives; reuse the metadata structure in `metadata/figshare_metadata.json`.

## Maintenance cadence | زمان‌بندی نگه‌داری
- Review content weekly and after major phases; add only non-sensitive language that aligns with the sanitized scope.
- Keep operational or network-specific data in private channels; do not add domains, IPs, configs, or bypass instructions here.
- Update metadata, checklist, and changelog whenever new public versions are prepared.

## Contact | ارتباط
For coordination, publication scheduling, or DOI linkage, use trusted channels only. Public inquiries can reference this sanitized documentation set.
